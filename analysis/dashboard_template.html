Collision Curve Analysis

Collision Curve Analysis Details

Collision curve analysis characterizes the relationship between pairwise sequence similarity and the empirical probability of hash collision. A well-behaved LSH family should exhibit monotonically increasing collision probability as similarity increases wherein highly similar sequences should frequently collide, while dissimilar sequences should rarely collide. This section reports empirical collision rates stratified across similarity bins, together with their variance across independent trials, for each hash family under evaluation. These curves serve as the primary first test for whether a given LSH scheme preserves the locality property under the mutation models and similarity metrics used while running the benchmark.

Binned Average Plot

Description : The binned average plot aggregates the sequence pairs into discrete similarity bins, showing the average collision rate for each bin. This provides a cleaner and clearer view of the underlying relationship between hash collision rate and similarity in comparision to the scatter plot.

This view quantifies the spread and skew of collision rates at each similarity level, allowing direct comparison of within-bin variance across the full similarity range, unlike the binned average which collapses each bin to a single value.

Threshold Based Similarity Search

Threshold Configuration & Analysis Details

Current Threshold: 0.95

Threshold-based similarity search evaluates the hash family's retrieval performance under a fixed similarity cutoff s_min. Given a query sequence, this test evaluates the hash family's ability to report all database sequences whose similarity to the query meets or exceeds s_min. Performance is quantified across recall, precision, and false positive rate, measured over a range of LSH configurations defined by the AND-OR amplification parameters (b,r) where b is the AND parameter and r is the OR parameter. This analysis exposes the fundamental trade-off between sensitivity and specificity inherent to probabilistic hashing, and identifies configurations that satisfy application-specific accuracy and runtime requirements for the dataset and distance metric under test.

Avg Recall vs Avg FPR

Description : This plot shows the trade-off between the Recall and False Positive Rate (FPR) across different (b,r)-configurations.The ideal operating point lies in the upper-left corner of the plot which correspond to high recall with low FPR. Configurations toward the lower-left sacrifice recall for specificity; those toward the upper-right achieve high sensitivity at the cost of acquiring more false candidates.

Avg Precision vs Avg Recall

Description : This plot illustrates the relationship between precision and recall. The ideal operating point lies in the upper-right corner of the plot which corresponds to high precision with high recall. Configurations toward the upper-left recover few neighbors but with high accuracy; those toward the lower-right achieve broad retrieval at the cost of returning more false candidates.

Min FPR per Recall Bin

Description : The binned minimum FPR plot partitions the recall range into discrete intervals and displays the lowest false positive rate achieved within each bin, for all (b,r) configurations. Rather than showing every operating point, this view isolates the optimal FPR attainable at each level of recall range, tracing the empirical lower bound of the retrieval trade-off curve. Bins where the minimum FPR remains low across high recall values indicate configurations where the hash family successfully maintains specificity without sacrificing sensitivity which is the target behavior for a well-tuned LSH scheme.
